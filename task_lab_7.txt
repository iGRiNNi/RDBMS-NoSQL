# Лабораторная работа. Работа с Elasticsearch

## Задание

Для СУБД **Elasticsearch** написать консольное приложение на **Java**, которое реализует следующий функционал:

1. **DAO-компоненты**, поддерживающие CRUD-операции для 3 связанных сущностей:

   * месторождение;
   * скважина;
   * запись добычи.

   Для каждой сущности должен быть создан отдельный индекс Elasticsearch.

   Связь между сущностями реализуется через идентификаторы, например:

   * `fieldId`;
   * `wellId`.

   Для демонстрации специфики Elasticsearch часть связанных данных должна быть денормализована внутри документов. Например, документ скважины может содержать не только `fieldId`, но и поля `fieldName`, `region`.

2. **Загрузка тестовых данных** через bulk-запрос.

3. **Запросы, демонстрирующие извлечение, обновление и удаление документов по условию фильтрации:**

   * извлечение активных скважин заданного типа и глубже заданного значения;
   * обновление статуса скважин по условию;
   * удаление записей добычи по условию.

4. **Показать разницу между операциями update и upsert:**

   * update существующего документа;
   * попытка update несуществующего документа;
   * upsert несуществующего документа.

5. **Показать разницу между частичным обновлением документа и полной заменой документа:**

   * частичное обновление через Update API;
   * полная замена документа через Index API с тем же `_id`.

6. **Продемонстрировать специфику Elasticsearch как поисковой системы:**

   * полнотекстовый поиск по описанию скважины;
   * поиск по нескольким текстовым полям;
   * нечёткий поиск с ошибкой в слове;
   * подсветка найденных фрагментов текста;
   * комбинированный `bool`-запрос с полнотекстовым поиском, фильтрацией, сортировкой и пагинацией.

7. **Продемонстрировать агрегирующие запросы:**

   * количество скважин по статусам;
   * средняя глубина скважин по типам;
   * суммарная добыча нефти по месторождениям.

## Требования к предметной области

Предметная область и способ работы с Elasticsearch выбираются студентом самостоятельно, но структура документов должна позволять наглядно продемонстрировать:

* CRUD-операции;
* фильтрацию документов;
* bulk-загрузку данных;
* разницу между `update` и `upsert`;
* разницу между частичным обновлением и полной заменой документа;
* полнотекстовый поиск;
* нечёткий поиск;
* подсветку найденного текста;
* сортировку;
* пагинацию;
* агрегирующие запросы.

## Основная цель работы

Лабораторная работа должна показать специфику Elasticsearch и его основное назначение — быстрый полнотекстовый поиск, фильтрацию, сортировку и агрегацию документов.
